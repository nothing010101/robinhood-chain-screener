Introduction

What is Ape Screener?

Ape Screener is a real-time token screener built for ape.store on Robinhood Chain (chain ID 4663). Every token launched on the platform is pulled into one live board — with market cap, volume, holder count, and on-chain wallet intelligence that most screeners don't show.

The core purpose is simple: help you see what's really happening with a token before you ape in, not after.

Chain: Robinhood Chain · #4663
Data source: ape.store API + Alchemy RPC (live on-chain)
App: app.apescreener.store
Main Screen

The Screener

The main board shows all tokens with a market cap of $7K or above, sorted by market cap descending by default. Tokens below $7K are still reachable via CA search.

📊

Market Cap Filter (MC ≥ $7K)

Filters out very new or illiquid tokens from the default view. Keeps the board readable without hiding anything — you can still search below-threshold tokens by contract address.

🔍

CA Search

Paste any contract address into the search bar to pull up that token instantly, regardless of MC. Useful for tokens freshly launched or still under $7K.

⚡

Live Data

Price and market cap are fetched directly from ape.store on every page load — no cache, no stale numbers. What you see is the current state of the chain.

Token Detail

Token Page

Click any token on the screener to open its detail page at app.apescreener.store/token/4663/[address]. The page shows full token metadata, market stats, and two on-demand analysis tools.

🧬

Bundle Analysis

Checks whether multiple early buyers share a common ETH funder — a signal of coordinated wallet setup. Triggered on click to save RPC quota.

🔗

Funding Trace

Follows the earliest incoming ETH transfer to the deployer wallet, showing who funded the dev before launch. Cached after the first lookup.

Feature

Bundle Detection

When you click Check Bundlers on a token page, the screener fetches the first 30 unique buyers and traces the earliest ETH funder for each wallet. Buyers who share the same funder are grouped as a "bundle."

1

Fetch early buyers

The first 30 wallets that received the token after launch are collected in chronological order.

2

Trace ETH funders

For each buyer wallet, the earliest incoming ETH transfer is found — this is who loaded the wallet before it started buying.

3

Group by shared funder

Wallets with the same ETH source are grouped. Groups of 2 or more are flagged as potential bundles.

4

Check sell / hold status

Each bundled wallet's current token balance is fetched live to show whether they still hold or have sold.

Bridge suppression: funders with a very high fan-out (30+ wallets funded) are treated as bridges or relayers and excluded from bundle warnings to reduce false positives.
Feature

Hold Percentage

Each bundled wallet shows its current share of total supply. All ape.store tokens launch with a fixed supply of 1,000,000,000 (1B) tokens, so the percentage is calculated directly:

Formula: (current balance ÷ 1,000,000,000) × 100
Example: 13,434,941 tokens → 1.34%

Balance is fetched live via a single eth_call per wallet at the moment you click Check Bundlers. It is never cached, so the number always reflects the current state.

⚠ 1.34% holding ⚠ 2.80% holding ✓ sold

A bundle where all wallets have already sold is a lower immediate risk than one where multiple wallets are still holding a combined 5–10% of supply.

Feature

Funding Trace

The Funding Trace section on a token page shows the earliest incoming ETH transfer to the token deployer wallet. This is not who the dev is — it is who funded the wallet that launched the token.

Why it matters: the same funder appearing behind multiple different token deployments is a strong signal of a serial operator — a dev running many launches from wallets they control, funded from a single source.

Results are cached permanently in the database after the first lookup, so repeated views are instant and don't consume Alchemy quota.

Note: if the dev wallet was funded via a bridge (Relay, Owlto, etc.), the trace will show the bridge relayer address rather than the real funder. Bridge addresses are flagged and suppressed where known.
FAQ

Common Questions

❓

Why is my token not showing on the screener?

Tokens below $7K market cap are filtered from the default view. Paste the contract address into the CA search bar to find it directly.

❓

Does "bundle detected" mean the token is a scam?

Not necessarily. It means multiple early wallets share a common ETH source. This could be a dev distributing to friends, a coordinated team, or genuine bundling. Use it as one signal alongside others, not a verdict.

❓

How fresh is the data?

Market cap and price are fetched live on every page load. Bundle analysis and funding trace are cached after the first check — balances within bundle analysis are always live.

❓

What chain does Ape Screener support?

Currently Robinhood Chain (chain ID 4663) only — the chain where ape.store's token factory is deployed.
